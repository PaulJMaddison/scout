# Universal Context Layer v2.0.0

Universal Context Layer v2.0.0 is the productisation release: the project moves from a local executive demo into an open-core context infrastructure product with a marketing site, self-hosted admin console, integration APIs, SDK scaffolds, blueprint import, SaaS-control-plane foundations, and a documented customer-owned data-plane architecture.

## Highlights

- Repositioned UCL as context infrastructure, not primarily an AI app.
- Added a clearer marketing website for platform, use cases, integrations, open core, pricing/deployment, docs, demo, and FAQ pages.
- Expanded the self-hosted admin console with organisation, workspace, users, API clients, usage, connectors, webhook events, blueprint imports, governance, audit export, and licence status screens.
- Added REST API v1 endpoints alongside GraphQL so systems can consume context without adopting GraphQL.
- Added API client authentication with key hashing, rotation, revocation, and last-used tracking.
- Added webhook/source-event ingestion with signature validation, idempotency, event storage, recompute triggers, dead-letter handling, and audit events.
- Added connector catalogue foundations with open-core connectors and explicit enterprise/vendor placeholders.
- Added AI-assisted blueprint JSON import without calling external AI APIs.
- Added billing and usage metering foundations without integrating Stripe, Paddle, or another payment provider.
- Added TypeScript and .NET SDK scaffolds for context consumers.
- Added PostgreSQL hosted-mode configuration, Docker packaging, Render deployment guidance, and local SQLite setup preservation.
- Added control-plane/data-plane documentation and local licence status foundations.

## Product Positioning

UCL turns existing business data into trusted semantic context that any AI tool, workflow, application, report, copilot, or agent can use. Customers do not need to use UCL's AI demo, and they do not need to move raw operational data into a hosted service.

The AI sales playground remains an example consumer. The same context layer can support internal copilots, support automation, CRM AI features, customer success, onboarding, reporting, workflow automation, third party agents, and internal business applications.

## Open Core Boundary

The public repo includes useful open-core functionality: local demo, self-hosted backend, admin console, GraphQL, REST, SDK scaffolds, generic connectors, mock connectors, blueprint import, event ingestion, audit, provenance, and extension interfaces.

The public repo does not include paid enterprise connector implementations, SSO/SAML, live billing-provider integration, hosted SaaS control-plane implementation, private cloud deployment packs, customer-specific mappings, compliance reports, or SLA support tooling.

# KynticAI Scout v2.4.0

KynticAI Scout v2.4.0 is a first paid pilot readiness release for the public open-core data plane. It keeps the paid/private implementation boundary intact and does not claim complete self-serve SaaS readiness.

## Paid Pilot Materials

- Added a first paid pilot one-pager for buyer conversations.
- Added a pilot SOW template with scope, objectives, source systems, semantic attributes, connector assumptions, deliverables, responsibilities, security/privacy, acceptance, support, exclusions, change control, pricing placeholders, and legal review disclaimer.
- Added legal and support starter templates with customer-owned data-plane assumptions and solicitor review disclaimers.

## Production Rehearsal

- Added production install rehearsal documentation.
- Added PowerShell and Bash rehearsal scripts that check production-style settings, PostgreSQL, demo fallback, strong signing key, persistent Data Protection keys, seed data, migration path, backup/restore commands, endpoint smoke checks, log/telemetry locations, and machine authentication route.
- The rehearsal prints exact commands and warnings instead of pretending a production deployment has succeeded.

## Connector Proof

- Added a first real connector proof using the generic SQL connector.
- Added sample connector configuration and sample SQL source data.
- Strengthened integration coverage so the SQL connector proof verifies source data in, selector mapping, context fact output, context snapshot output, service/API-shaped context lookup, provenance metadata, and recompute audit trail.

## Verification

- Public targeted connector proof: `dotnet test .\tests\KynticAI.Scout.IntegrationTests\KynticAI.Scout.IntegrationTests.csproj --filter "FullyQualifiedName~SqlTableConnector_ReadsCurrentDatabaseRow_AndProducesContextSnapshot"`.
- Production rehearsal script ran locally with production-style environment variables and completed configuration checks and build. Live API health checks were documented as unavailable because no API process was running.

## Boundary

The public repo remains open-core only. Real paid vendor connector implementation, private enterprise modules, private cloud control-plane implementation, customer secrets, local runtime artefacts, generated licences, and raw customer data remain outside this repository.

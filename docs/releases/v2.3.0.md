# KynticAI Scout v2.3.0

Release date: 2026-05-12

KynticAI Scout v2.3.0 is a public open-core/data-plane hardening release for paid pilot readiness. It does not make the product a complete self-serve SaaS, and it does not move private enterprise or cloud implementation code into the public repository.

## Public Open-Core Changes

- Added separate webhook signing secrets for production webhook verification, distinct from API client secrets.
- Added create, list, rotate, and revoke REST operations for webhook signing secrets.
- Stored only hashed webhook secret values.
- Added timestamp and event ID replay checks for webhook-secret-authenticated event ingestion.
- Added a provider adapter seam for future signature formats without adding private provider implementations.
- Added public-safe governance hook interfaces and default no-op behaviour for private enterprise policy packages.
- Formalised the canonical API scope set, including `context:write` as an official write scope.
- Added local pilot readiness scripts and a paid pilot runbook.

## Boundary

The public repository contains only public-safe code: open-core data-plane behaviour, interfaces, default/no-op implementations, placeholder connector catalogue metadata, docs, scripts, and fictional examples.

Paid enterprise connector implementations, private SSO/SAML/SCIM assertion handling, advanced governance policy implementation, vault integrations, deployment packs, SLA tooling, hosted account/billing/licence/support portals, and cloud/control-plane implementation remain private commercial work.

## Operational Notes

`VITE_DEMO_FALLBACK=true` remains local-demo only. Paid pilot and production-style deployments should use `VITE_DEMO_FALLBACK=false`, `Bootstrap__SeedDemoData=false`, PostgreSQL connection strings, explicit signing keys, and customer-controlled connector credential stores.

The customer data plane remains the owner of operational data by default. Raw customer records, connector credentials, context facts, prompt context packages, message bodies, documents, attachments, and raw analytics payloads should not be sent to a hosted control plane by default.

## Verification Summary

Release verification should include the .NET build and focused backend tests, frontend lint/build, TypeScript and .NET SDK checks when package metadata changes, pilot readiness dry run, public forbidden-code scan, secret/runtime artefact scan, no-GitHub-Actions check, and GitHub visibility confirmation.

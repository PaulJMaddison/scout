# KynticAI Scout v2.2.0

Release date: 2026-05-12

KynticAI Scout v2.2.0 is an additive public open-core release aligned with a private enterprise connector-suite release.

## Public Open-Core Release

The public repository continues to ship the self-hosted data-plane foundations, local demo, REST and GraphQL APIs, SDKs, connector interfaces, generic connectors, and public documentation. It does not ship paid enterprise connector implementations or hosted cloud/control-plane implementation code.

The release also moves the commercial story towards a supported paid pilot: customer-owned data plane first, implementation-led onboarding, and clear boundaries around what is available now versus future cloud/control-plane work.

New public-safe catalogue placeholders cover:

- Microsoft 365 / Outlook and Gmail / Google Workspace
- Slack and Microsoft Teams
- Outlook Calendar and Google Calendar
- Segment, Amplitude, Mixpanel, and PostHog
- Jira and Linear
- Confluence, Notion, SharePoint, and Google Drive

These entries are metadata only. They cannot be configured or executed as real connectors in the public repository.

## Safety Model

The aligned private enterprise connector model is metadata-first. Communication and knowledge connectors default to metadata-only ingestion. Message bodies, chat bodies, calendar descriptions, issue descriptions, document content, and attachments require explicit customer opt-in in enterprise deployments. Raw customer operational data is not sent to the hosted/cloud control plane by default.

`VITE_DEMO_FALLBACK=true` is local-demo only. Customer and production-style deployments must use `VITE_DEMO_FALLBACK=false` so API failures are visible and never replaced by mock data.

## Hardening

This release includes public data-plane hardening around REST and GraphQL scope enforcement, correlation-friendly permission errors, and webhook safety documentation.

## Commercial Readiness

New commercial-readiness documents:

- [Paid pilot](../paid-pilot.md)
- [Production install checklist](../production-install-checklist.md)
- [Anonymised ERP platform pattern](../anonymised-erp-platform-pattern.md)

The anonymised ERP platform pattern explains recent implementation experience where a semantic data plane over existing operational data helped a new web platform and AI-enabled workflows consume business meaning rather than raw records. It does not name the customer, disclose confidential architecture, or claim a formal case study.

Pricing and commercial copy now position the product as sellable through discovery workshops, starter pilots, production pilots, and enterprise rollout design. The public repo still does not include live card payment, hosted account management, support portal, licence portal, or complete self-serve SaaS operations.

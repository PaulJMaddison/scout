# KynticAI Scout v2.5.0

KynticAI Scout v2.5.0 added an earlier company website preview path for public sales, marketing, and buyer education while preserving the full downloadable product path in the repository. That preview path was later retired from the open-source repo in v2.8.0.

## Highlights

- Public web app that can be built without the backend, databases, Docker, SQLite, PostgreSQL, or local setup scripts.
- Fictional Northstar Logistics / User 123 / Avery Stone story showing raw operational signals, selector logic, semantic facts, AI recommendations, provenance, and workflow outcomes.
- Dedicated website build and preview scripts for `apps/web`.
- Production web build and preview workflow for the public site.
- Generated fallback files for website publishing.
- Documentation explaining how the public website differs from the full local product.
- Laptop and mobile screenshot captures for website docs.

## Product Boundary

The website preview path was a brochure, sales, and marketing walkthrough. It did not call the live backend and did not prove connector execution, authentication, tenant isolation, database migrations, or production operations.

The full product remains available in the public repo as the functional React application, backend, REST and GraphQL APIs, SDKs, selector engine, seeded demo, and local setup path.

## Commercial Boundary

This release kept the public repo as open-core product, documentation, demo, and customer-owned data-plane positioning. It did not add paid vendor connector implementation code, customer operational data processing in the website preview, or complete self-serve SaaS operations.

# Universal Context Layer v2.5.0

Universal Context Layer v2.5.0 adds a static GitHub Pages brochure/demo site for public sales, marketing, and buyer education while preserving the full downloadable product path in the repository.

## Highlights

- Static GitHub Pages demo that can be built without the backend, databases, Docker, SQLite, PostgreSQL, or local setup scripts.
- Fictional Northstar Logistics / User 123 / Avery Stone story showing raw operational signals, selector logic, semantic facts, AI recommendations, provenance, and workflow outcomes.
- Dedicated static build and preview scripts for `apps/web`.
- GitHub Pages base-path support for `/universalcontextlayer/`.
- Generated `index.html` and `404.html` fallback files for Pages publishing.
- Documentation explaining how the static site differs from the full local product.
- Laptop and mobile screenshot captures for the static demo docs.

## Product Boundary

The static site is a brochure, sales, and marketing walkthrough. It does not call the live backend and does not prove connector execution, authentication, tenant isolation, database migrations, or production operations.

The full product remains available in the public repo as the functional React application, backend, REST and GraphQL APIs, SDKs, selector engine, seeded demo, and local setup path.

## Commercial Boundary

This release keeps the public repo as open-core product, documentation, demo, and customer-owned data-plane positioning. It does not add paid vendor connector implementation code, customer operational data processing in the static site, or complete self-serve SaaS operations.
